## Workflows

| Command | What it does |
|---------|-------------|
| `/analyze` | Evaluate a problem/PRD, research gaps |
| `/define` | Create JTBDs and user stories from research |
| `/plan` | Design architecture and sprint plan |
| `/build` | Execute sprint stories (routes to frontend/backend/mobile) |
| `/review` | Full QA pipeline: test, review, audit, evaluate, optimize |
| `/hotfix` | Quick bug fix with lightweight QA |
| `/code-review` | Review existing code or PR |
| `/new-project` | Initialize a new project (you're reading the output) |

## Orchestration Rules

1. Start every non-trivial task in **plan mode** — write plan to `tasks/todo.md` first
2. **Commit after each completed story** — small, focused commits
3. Update `docs/PROJECT_KNOWLEDGE.md` after each feature
4. Consult `tasks/lessons.md` at the start of each session
5. Use `/compact` at 50% context usage
