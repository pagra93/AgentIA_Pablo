# Bad PRD Example: Warehouse Efficiency App

## What's Wrong

### Problem Statement
"We need to modernize the warehouse receiving process with a digital solution that integrates barcode scanning, real-time inventory sync, and automated reports."

**Problems**: Prescribes the solution. No actual problem described. "Modernize" is not a problem.

### Metrics
"Improve efficiency and reduce errors."

**Problems**: No baseline. No target. No unit. "Reduce errors" from what to what?

### Process
"The current process is manual and slow."

**Problems**: No AS-IS steps. No TO-BE. No actors. "Manual and slow" is opinion, not observation.

### Additional Issues
- "Use REST API with SAP integration" → technical prescription
- "Dashboard with real-time updates" → UI prescription
- No field observations or user quotes
- No Gemba walk data

## Quality Guard Score: ~3/10
- D1 (Completeness): 2/10 — no metrics, no observations
- D2 (Process): 1/10 — no AS-IS/TO-BE
- D3 (WHAT/HOW): 4/10 — multiple solution prescriptions
